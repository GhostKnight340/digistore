NOTES

Layout decisions

Why it's built this way — for the Claude Code implementation pass
Six destinations, not twenty
Catalogue, Orders and Settings each collapse a cluster of old pages into a sub-grouped section. Homepage Editor is pulled out of the sidebar into a prominent header button since it's a distinct mode, not a list.
Save is never below the fold
Every editor (product, settings, email) puts actions in a sticky bar pinned to the bottom of its own scroll region, with a live "unsaved / saved" status. Destructive actions live top-right, away from Save.
Variants are first-class
The product workspace is a 3-pane split: list, editor, and an always-visible variant rail with per-variant stock and price — no hidden accordions to expand before editing.
Orders get a full page
Payment review, code entry, delivery, timeline and emails all sit on one split-view page. Confirm / Reject / Request-proof stay in the header strip so they're reachable regardless of scroll.
Inventory reads like a human
Grouped by product then variant with stock bars and color-coded counts, plus a top alert strip for out-of-stock and low-stock — not a flat SKU database table.
One consistent system
Geist + Geist Mono, a single blue accent, #2EA067/#E8A838/#E05C5C for success/warning/danger, thin rgba(255,255,255,.07) borders and 14px card radii applied everywhere. Toggles, badges and tables are shared primitives.
