# Rules for Claude Code
# read at the start of every session

1. Smallest change that works.
2. Match the style of the file you're in.
3. Never say "done" until the tests pass.
4. Unsure? Ask — don't invent an API.
5. Plan first — list the files before editing.
6. Search before read — don't load whole files.
7. Show the diff before big changes.
